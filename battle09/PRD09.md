## 一、产品概述

### 1.1 产品定位

BattleMap v0.9.0 是一个面向**金融租赁行业**的产业客户作战指挥平台，定位为「**GIS（地理信息）+ 产业大数据 + 协同指挥」的投行级资产管控与创利沙盘**。

本版本整合了 PRD.md 的精细化数据管理能力和 PRDbygemini.md 的战略指挥视角，新增统帅驾驶舱和战区指挥台，为高管和团队长提供全局作战视图。

### 1.2 目标用户

| 用户角色 | 核心场景 |
|---------|---------|
| 高管/事业部总监 | 统帅驾驶舱查看全局 KPI、红绿灯预警网、渗透率分析 |
| 团队长/组长 | 战区指挥台进行框选派单、轨迹追踪、网格管理 |
| 客户经理 | 个人工作台执行工单、管理商机、数据补录 |
| 运营分析人员 | 宏观态势面板查看产业地图、产业链分析 |
| 风控人员 | 防雷预警监控、风险企业追踪 |

### 1.3 核心价值

- **全局指挥**：一张地图看清全国产业格局、园区分布、风险态势
- **战略决策**：统帅驾驶舱提供北极星 KPI 和红绿灯预警网
- **高效协同**：战区指挥台支持地图框选派单和团队轨迹追踪
- **数据驱动**：支持 18 种数据类型的 CSV/JSON 导入，导入后全系统联动

## 三、功能规格

### 3.1 统帅驾驶舱（`/commander`）

#### 3.1.1 北极星 KPI 看板

**功能描述**：高管视角的核心指标展示，包括：

| 指标 | 说明 | 数据来源 |
|------|------|---------|
| 实际投放额 | 累计设备投放规模（亿元） | adminData['投放'] |
| 融资租赁余额 | 融资租赁在租余额（亿元） | adminData['融资租赁'] |
| 平均渗透率 | 园区渗透率平均值 | penetrationData |
| 设备IOT在线率 | IOT设备在线比例 | firms.iotStatus |
| 逾期敞口 | 逾期本息合计（亿元） | firms.indicators.overdueAmount |

**关键交互**：
- 指标卡片支持点击下钻到详细视图
- 趋势箭头显示环比变化
- 紧急预警闪烁提示

#### 3.1.2 红绿灯资产预警网

**功能描述**：全国企业节点按风险状态着色，闪烁高亮高危企业。

**节点状态**：
- **绿色（健康）**：风险正常，IOT 在线，无预警
- **黄色（关注）**：风险上升，连续亏损，信用评分下降
- **红色闪烁（高危）**：逾期超过 30 天、IOT 离线、中登网新增抵押

**数据来源**：`riskAlerts`（风险预警）+ `firms`（企业状态）

#### 3.1.3 区域视角热力图

**功能描述**：按省份展示产业热力分布，支持指标切换。

**支持的指标**：
- 投放规模（亿元）
- 融资租赁余额（亿元）
- 渗透率（%）
- 不良率（%）
- 活力指数（0-100）

#### 3.1.4 防雷预警监控

**功能描述**：统一展示所有未确认的风险预警。

**预警类型**：
- **逾期预警**：逾期天数 ≥ 30 天
- **IOT离线**：设备连续 48 小时无数据
- **中登网抵押**：新增动产抵押登记
- **信用评分下降**：连续 3 个月评分下降

---

### 3.2 战区指挥台（`/battlefield`）

#### 3.2.1 模式切换

支持三种模式的切换：
- **框选派单**：在地图上绘制多边形，选择区域内的园区和企业，批量派发工单
- **鹰眼轨迹**：查看客户经理的拜访轨迹，按时间线展示
- **网格渗透率**：按园区显示渗透率达标热力

#### 3.2.2 框选派单

**功能描述**：在地图上绘制多边形区域，选中园区和企业后批量派发工单。

**操作流程**：
1. 点击「框选派单」模式
2. 在地图上点击并拖拽绘制多边形
3. 选中的园区和企业高亮显示
4. 点击「派发工单」，填写工单信息
5. 选择执行人（可多选）
6. 确认派发

#### 3.2.3 鹰眼轨迹追踪

**功能描述**：查看客户经理的拜访轨迹和打卡记录。

**数据来源**：`visitTrails`（拜访轨迹）+ `visitRecords`（拜访记录）

**展示内容**：
- 轨迹路线（带方向箭头的飞线）
- 打卡节点（圆点标记）
- 拜访详情（点击节点查看纪要）
- 行程统计（总里程、企业数）

#### 3.2.4 网格渗透率监控

**功能描述**：按园区显示渗透率达标情况。

**状态颜色**：
- **绿色（超额完成）**：渗透率 > 目标渗透率
- **黄色（达标）**：渗透率 ≈ 目标渗透率（±5%）
- **红色（待提升）**：渗透率 < 目标渗透率

## 四、数据模型

沿用 PRD.md 规格，保持 18 种数据类型不变。详见 PRD.md 第四章。

### 4.1 新增数据类型

| 类型 | 字段 | 说明 |
|------|------|------|
| `riskAlert` | 风险预警 | 逾期/IOT/抵押/评分预警 |
| `visitTrail` | 拜访轨迹 | 经理轨迹点和里程统计 |
| `penetrationData` | 渗透率数据 | 园区/产业链渗透率统计 |

### 4.2 数据一致性保证

```
importer.ts → importFile(objectType, text, fileType)
    ↓
parseXxx() 解析数据
    ↓
_db.{table} = parsedData  [写入内存数据库]
    ↓
syncToStore(objectType, data)  [同步到 Zustand Store]
    ↓
useAppStore.setState({ [dataType]: data })  [触发 React 重渲染]
    ↓
rebuildIndex()  [重建全局索引]
    ↓
所有页面通过 useAppStore 感知变化并重渲染
```

## 七、非功能需求

### 7.1 性能

- 首屏加载：GeoJSON 按省份懒加载，首屏仅加载全国底图
- ECharts 图表：ResizeObserver 监听容器变化自动 resize
- 大数据量：虚拟列表（如果单类型数据 > 1000 条需考虑）

### 7.2 兼容性

- 目标浏览器：Chrome/Edge 最新版
- 响应式：最小支持 1280px 宽度（桌面端产品）

### 7.3 数据一致性

- **核心原则**：所有业务数据必须流经 Zustand store
- **导入即生效**：导入操作必须同步更新 store，否则页面不响应
- **索引重建**：每次数据变更后自动重建 DataIndex 全局索引
