江苏金租“作战地图”系统需求规格说明书 (PRD/SRS v1.0)1. 引言1.1 编写目的本文档旨在明确江苏金租“业务作战地图”系统的产品定位、业务边界、核心功能与系统架构。作为研发、测试、UI/UX及业务干系人的需求基线，指导后续的系统开发与交付。1.2 产品背景与定位江苏金租业务涵盖医疗健康、清洁能源、高端制造等实体产业，具备“重资产、大金额、极度依赖线下尽调与跨部门协同”的特点。 本系统定位为**“GIS（地理信息）+ 产业大数据 + 协同指挥”的投行级资产管控与创利沙盘**。旨在解决业务员拓客盲目、现场尽调过程难管控、团队协同效率低，以及高管无法实时掌控前线战况与风险资产地理分布的核心痛点。1.3 核心业务价值洞察与下钻：宏观把控产业与园区热力，计算金租市场渗透率。商机与破冰：外接企信数据挖掘需求，通过存量客户关系图谱实现智能引荐。指挥与协同：战区网格化管理，支持多边形框选派单。单兵与促单：移动端LBS雷达防盲跑，提供租金/IRR精算器辅助现地逼单。风控与防雷：构建“内部逾期 + 外部中登网 + 物联IOT”三位一体的地图防雷网。2. 用户角色与使用场景角色名称核心痛点核心 User Story (用户故事)高管/事业部总监宏观洞察弱、风险有盲区、战况不透明希望在统帅大屏上切换“产业/区域”视角，查看渗透率、实际投放额、折算资金成本投放额，并能对异常标的进行下钻穿透看风险。团队长/小组长派单盲目无依据、外勤过程黑盒希望在地图上框选园区批量派发工单，并能直观看到组员结合自有APP打卡生成的日勤跑动轨迹与现场纪要。一线客户经理拓客如盲人摸象、现场促单缺武器希望手机端能基于LBS推荐周边3公里潜客，自动提供同园区/同产业链存量客户的“熟人引荐路径”，并使用移动精算器。厂商协作/中后台(二期规划)信息差、审批时效慢作为标签录入，数据由客户经理上传。预留审批与风控API接口，未来支撑“虚拟战队”联合作战。3. 总体架构设计3.1 核心架构分层系统采用“三端展现、三大引擎、四向对接”的轻量化数据驱动架构：表现层 (多端适配与千人千面)统帅大屏 (PC/大屏)：全局产业渗透罗盘、产能/逾期风险热力图。战区指挥台 (PC)：辖区客户落点地图、地图框选派单、组员鹰眼轨迹。单兵作战雷达 (移动端H5/企微)：LBS周边雷达、触达引荐图谱、租金精算器。应用层 (核心计算引擎)多维渗透率引擎 (计算园区/产业链渗透率)。智能图谱指引引擎 (计算最短引荐路径)。网格与轨迹渲染引擎 (解析自有APP打卡点拟合连线)。数据底座GIS空间数据库 (PostGIS等) + 关系图谱数据库 (Neo4j等) + 缓存。3.2 系统内外对接清单本系统重在“情报处理”，不生产基础打卡数据。自有打卡APP (单向拉取)：获取打卡流水（人、时间、经纬度、关联对象、沟通纪要），用于鹰眼轨迹绘制。核心业务系统 (T+1拉取)：获取存量客户、起租状态、实投金额、内部逾期天数。企信/招投标大数据 (外部API)：天眼查/企查查（获取规上企业白名单、股权/供应链关系）、招投标网（获取采购商机）。外部风控接口 (实时查询)：中登网（动产抵押）、人行征信、IOT设备云平台。商业地图平台：高德/百度地图政区与园区瓦片底座。4. 核心功能模块需求4.1 统帅沙盘驾驶舱 (高管端)双轴观测视角切换：支持“产业视角（如医疗、绿能）”与“区域/战区视角”无缝切换，地图重绘热力分布。北极星 KPI 看板：实际投放额 / 折算资金成本的投放额 (EVA视角)。核心设备 IOT 在线开机率。内部逾期暴露敞口。红绿灯资产预警网：地图节点分色显示。健康(绿)、跟进中(蓝)、高危(红灯闪烁)。极限穿透探针 (Drill-down)：点击地图节点弹出抽屉视图。产能穿透 (钱包份额)：展示包含“金租投放、银行债权、股权融资、同业融资”的全口径融资结构饼图。风险穿透 (防雷网)：联合展示内部逾期天数、中登网新增抵押预警、设备IOT离线状态及征信涉诉情况。4.2 战区指挥台 (团队长端)多边形框选派单：在电子地图上圈定特定园区或多边形区域，将区域内的“沉睡/公海商机”一键分配给指定组员。鹰眼轨迹追踪：基于组织架构，拉取自有APP打卡数据，在地图上绘制组员当日/历史的拜访移动路线图，点击节点可查看《拜访纪要》。网格渗透率监控：查看所辖战区内各个细分园区的渗透率达标情况。4.3 单兵作战雷达 (一线业务员端)同园区周边雷达 (LBS)：外勤出差时，基于GPS定位，地图显示3公里内近期有招投标动作或企信资质良好的潜客，减少盲跑。智能引荐图谱 (破冰引导)：系统自动计算目标客户与现有起租客户的关联（如：同园区邻居、上下游供应链），提示业务员通过存量客户进行“熟人引荐”。移动端精算器：内置符合金租业务逻辑的计算器，输入本金、期数、利率、保证金，秒出等额本息/本金还款表与 IRR，支持一键截图发给客户。厂商标签体系：允许业务员为商机打上“厂商租赁/直租”及具体厂商渠道的标签，沉淀渠道数据。5. 核心指标与算法定义5.1 渗透率模型 (Penetration Rate)公式：某维度渗透率 = 核心系统该维度内已起租客户数 / 企信API获取的该维度规上目标企业总数维度：支持按“产业园区”或“特定产业链”计算。5.2 钱包份额模型 (Wallet Share)公式：客户总融资盘 = 我司实际投放额 + 同业金租融资额 + 银行与债券融资额 + 股权融资额用途：指导高管与业务员识别杠杆健康度及继续扩容授信的挖潜空间。6. 核心业务闭环流程 (状态机)地图商机节点遵循以下生命周期演进：[沉睡] 灰色节点：系统爬取外部招投标或企信数据形成的新增白名单企业。[网格] 蓝色节点：团队长在地图上通过多边形框选，派发给指定一线业务员。[破冰] 路径生成：系统生成基于存量客户的智能引荐图谱。[交火] 紫色节点：业务员到达现场，使用自有APP完成GPS打卡并录入纪要，节点变紫。[落定] 绿色节点：签约起租，实投金额回传，节点变绿，园区渗透率自动提升。[防雷] 红色节点：起租期间一旦触发中登网重复抵押、断交租金或IOT离线，即刻变红高频闪烁。7. 非功能性需求权限控制：基于 OA 组织架构树，实现严格的数据隔离（小组长仅看本组，高管看全局/管辖事业部）。性能要求：地图渲染万级以上的散点需使用 WebGL/Canvas 技术（如 ECharts），确保缩放、拖拽不卡顿。穿透查询响应时间需小于 2 秒。安全要求：涉及企业敏感融资数据与客户征信，数据传输需全程加密，支持水印防泄密。
